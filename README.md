# Premier Tree Specialists — Lead Intake Dashboard

Automated lead intake, AI response generation, and CRM sync for Premier Tree Specialists LLC.

---

## Running Locally

You need **two terminals** — one for the backend, one for the frontend.

### Terminal 1 — Backend (Hono API)

```bash
# From project root
npm install
npm run dev
# → http://localhost:3000/api
```

### Terminal 2 — Frontend (Vite + React)

```bash
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

Open **http://localhost:5173** in your browser. The frontend automatically proxies all `/api` requests to the backend on port 3000.

> **No API keys needed for local dev.** All external services (OpenRouter, ArboStar, SendGrid) run in mock/log mode when keys are absent.

## Environment Variables

All variables in `.env` are optional for local development:

| Variable | Purpose | Required for |
|---|---|---|
| `DATABASE_PATH` | SQLite file path (default: `./leads.db`) | Always |
| `OPENROUTER_API_KEY` | Real AI responses via Claude | Production |
| `ARBOSTAR_COMPANY_ID` | ArboStar CRM company ID | Production |
| `ARBOSTAR_API_KEY` | ArboStar API key | Production |
| `SENDGRID_API_KEY` | Email sending | Production |
| `FROM_EMAIL` | Sender address | Production |
| `AGENT_PHONE_API_KEY` | SMS/iMessage sending | Production |
| `AGENT_PHONE_NUMBER` | Business phone number | Production |

## API Endpoints

### Leads
| Method | Path | Description |
|---|---|---|
| GET | `/api/leads` | All leads (`?status=review` to filter) |
| GET | `/api/leads/:id` | Single lead + ArboStar payload + audit log |
| GET | `/api/leads/meta/stats` | Dashboard stats |
| POST | `/api/leads/ingest/form` | Ingest website form submission |
| POST | `/api/leads/ingest/email` | Ingest LSA or AnswerForce email |

### Actions
| Method | Path | Description |
|---|---|---|
| POST | `/api/respond/:id/approve` | Approve + send (optional edited draft in body) |
| POST | `/api/respond/:id/reject` | Reject lead |
| PATCH | `/api/respond/:id/draft` | Update response draft |
| GET | `/api/respond/:id/log` | Full audit log for a lead |

## Test with curl

```bash
# Ingest a website form lead
curl -X POST http://localhost:3000/api/leads/ingest/form \
  -H "Content-Type: application/json" \
  -d '{"name":"Brian K","email":"brian@gmail.com","phone":"216-555-0100","zip":"44121","serviceType":"Tree Trimming","message":"Big oak near power lines"}'

# Ingest an AnswerForce email
curl -X POST http://localhost:3000/api/leads/ingest/email \
  -H "Content-Type: application/json" \
  -d '{"from":"af@example.com","subject":"After Hours","body":"Call at 8:44 PM, Name: Tom E, Phone: 440-555-0271, Details: Tree leaning on house after storm","source":"AnswerForce"}'

# Approve a queued lead (replace LEAD_ID with real id from above)
curl -X POST http://localhost:3000/api/respond/LEAD_ID/approve \
  -H "Content-Type: application/json" \
  -d '{"actor":"jane"}'
```

## Build for Production (before Railway deploy)

```bash
# From frontend/ — builds React app into ../public/
cd frontend && npm run build

# From project root — compiles TypeScript
npm run build

# Now deploy
railway up
```

---

## Deploy to Railway

```bash
npm install -g @railway/cli
railway login
railway init
railway volume add --mount /data
railway variables set DATABASE_PATH=/data/leads.db
# Set remaining vars in Railway dashboard
railway up
```
